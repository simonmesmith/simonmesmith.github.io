* Certain types of development work—especially translating clear requirements into code—should be _particularly_ amenable to automation, since programming languages are less ambiguous than spoken languages such as English, and there is _so_ much code online for machines to learn from, such as on Stack Overflow and GitHub.
* For example, researchers have used GitHub to create a database of Python code and its associated documentation, then trained an AI on this database. They believe that in future this could allow machines to automatically provide documentation for given code or code for given documentation.[^1]
* Startups are emerging that promise to automate aspects of software development. [Crane][2], for example, uses AI to transform ideas and design into prototypes and code. [Heek][3] is a chatbot that lets users create a website by simply talking to it.

[^1]:	[A parallel corpus of Python functions and documentation strings for automated code documentation and code generation][1]
